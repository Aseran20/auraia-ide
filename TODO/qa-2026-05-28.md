## Découvertes workflow (2026-05-28 PM)

| Découverte | Implication |
|---|---|
| `useEsbuildTranspile=false` dans la config — `npm run watch` fait QUE du type-check (noEmit) | Le vrai emit vers `out/` se fait avec `node build/next/index.ts transpile` (10s one-shot). **Loop optimal = edit → transpile (10s) → Ctrl+R via agent-browser.** Pas besoin de watch complet. |
| `watch-extensions` peut crash avec `ERR_CHILD_PROCESS_STDIO_MAXBUFFER` sur sub-task média | Non-bloquant pour les changes côté workbench. Si crash, juste relancer `node build/next/index.ts transpile` après chaque edit au lieu de `npm run watch`. |
| Le pattern `.disabled` pour neutraliser une base patch est validé par `prepare_vscode.sh:159` (glob `*.patch`) | Préférer ça à un user patch qui annule, quand la base patch ajoute du code mort pour Arclen (~50 LOC évitées dans le cas announcements). |
| `check-patches.sh` a un bug de cwd relatif (cherche `patches/user/X.patch` depuis le mauvais répertoire) | À fixer dans un futur batch. N'invalide pas les patches eux-mêmes. |
| Ctrl+R via `agent-browser press` déconnecte le CDP | Reconnect avec `agent-browser connect 9222` après reload, puis screenshot. |
